
# Настройка мониторинга логов и блокировки BitTorrent для Remnawave (Xray)

Этот репозиторий содержит инструкцию по настройке сбора логов и блокировки BitTorrent трафика на сервере с Remnawave (Xray). Вы узнаете, как подключить мониторинг логов с помощью специального скрипта и настроить автоматическую ротацию логов, чтобы не забить диск.

## Предварительные требования

*   Сервер с установленной Remnano (Xray).
*   Доступ к серверу по SSH с правами root или sudo.
*   Установленные `docker` и `docker-compose`.

---

## Пошаговая инструкция

### 1. Создание директории для логов на сервере

Первым делом необходимо создать папку, где будут храниться логи, и сделать её доступной для Docker-контейнера Remnanawave.

Выполните команды на сервере:

```bash
mkdir -p /opt/remnanode && cd /opt/remnanode
```

### 2. Подключение тома к контейнеру Remnawave

Чтобы Remnanawave (Xray) мог писать логи в созданную папку, нужно подключить её как том (volume) в Docker-контейнер.

1.  Откройте конфигурационный файл Docker Compose:
    ```bash
    nano /etc/remnanode/docker-compose.yml
    ```

2.  Найдите раздел `volumes:` внутри описания сервиса (обычно `xray` или `app`) и добавьте туда строку:
    ```yaml
    volumes:
      - "/var/log/remnanode:/var/log/remnanode" # Добавьте эту строку
    ```
    *Убедитесь, что соблюдены отступы (пробелы), как в примере ниже.*

3.  Сохраните файл (`Ctrl+O`, `Enter`) и выйдите из редактора (`Ctrl+X`).

4.  Перезапустите контейнер Remnawave для применения изменений:
    ```bash
    cd /etc/remnanode && docker compose down && docker compose up -d
    ```

### 3. Установка скрипта для мониторинга логов

Теперь установим специальный скрипт, который будет анализировать логи и взаимодействовать с файрволом для блокировки нежелательного трафика.

Запустите установку скрипта одной командой:

```bash
bash <(curl -fsSL git.new/install)
```

В процессе установки скрипт задаст вам два вопроса. Ответьте на них следующим образом:

1.  **Enter the path to the log file to monitor:**
    Введите путь до лог-файла, за которым нужно следить.
    ```
    /var/log/remnanode/access.log
    ```

2.  **Select firewall (iptables/nft):**
    Выберите тип файрвола. Если вы не уверены — выбирайте `nft`.
    *   Введите `nft` (рекомендуется для новых систем)
    *   Или `iptables` (для старых систем)

### 4. Редактирование конфигурации Xray

Теперь нужно настроить сам Xray (Remnano), чтобы он начал писать логи в нужный файл.

1.  Перейдите в веб-панель управления Remnawave.
2.  Откройте раздел редактирования конфигурации Xray.
3.  В самое начало конфигурационного файла (в объект `"log"`) добавьте следующие строки:

    ```json
    "log": {
        "access": "/var/log/remnanode/access.log",
        "error": "/var/log/remnanode/error.log",
        "loglevel": "error"
    }
    ```
    *Важно: Убедитесь, что перед этим объектом нет ничего, кроме открывающей фигурной скобки `{` всего конфига.*

### 5. Настройка блокировки BitTorrent

Чтобы запретить использование сервера для торрентов, добавим соответствующее правило маршрутизации.

1.  В том же конфигурационном файле Xray найдите массив **`"routing"`**, а внутри него — массив **`"rules"`**.
2.  Добавьте в массив `"rules"` новое правило (например, в самый конец списка, но перед запятой у предыдущего элемента):
    ```json
    {
      "protocol": ["bittorrent"],
      "outboundTag": "TORRENT",
      "type": "field"
    }
    ```

3.  Затем найдите массив **`"outbounds"`** и добавьте в него новый исходящий канал, который будет отправлять торрент-трафик в "никуда" (blackhole):
    ```json
    {
      "protocol": "blackhole",
      "tag": "TORRENT"
    }
    ```

4.  **Важно:** Не забудьте поставить запятую после предыдущего объекта в массивах `rules` и `outbounds`. Сохраните конфигурацию в панели управления.

### 6. Настройка ротации логов (чтобы диск не переполнился)

Логи имеют свойство расти и со временем могут занять всё свободное место на диске. Чтобы этого избежать, настроим автоматическую ротацию и сжатие старых логов.

Выполните на сервере следующую команду одной строкой. Она создаст файл конфигурации для системы `logrotate`:

```bash
sudo bash -c 'cat > /etc/logrotate.d/remnanode <<EOF
/var/log/remnanode/*.log {
    size 50M
    rotate 5
    compress
    missingok
    notifempty
    copytruncate
}
EOF'
```

**Что означают эти настройки:**
*   `size 50M`: Начинать ротацию, когда файл достигнет размера 50 МБ.
*   `rotate 5`: Хранить 5 последних ротаций (архивов), остальные удалять.
*   `compress`: Сжимать старые файлы в gzip.
*   `missingok`: Не выдавать ошибку, если лог-файла вдруг нет.
*   `notifempty`: Не ротировать пустой файл.
*   `copytruncate`: Копирует содержимое лога в архив и очищает оригинальный файл. Это безопасно для программ, которые постоянно держат файл открытым (как Docker/Xray).
